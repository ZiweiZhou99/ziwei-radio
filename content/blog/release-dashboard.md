# 用 vibe coding 做了一个发版管理看板

[→ 打开看板](http://www.zhouziwei.online/release-dashboard.html)

## 问题是怎么来的

我们团队管版本计划这件事，长期处于一种很混乱的状态。

发版相关的信息散落在三个地方：Confluence 上有需求文档和测试报告，石墨上有版本计划表，腾讯文档里有另一份发版记录。三个平台，没有一个是权威的，每次要查「S2 现在是什么版本」或者「E1 上个版本发了什么」，都需要挨个翻。

开发同学问我的时候，我经常自己也要找一会儿。

## 做了什么

花了一个周末，用 AI 辅助开发的方式做了一个专属的发版管理看板。

核心功能很简单：
- 各机型（S6、S2、S1、E1）的当前版本状态一览
- 发版计划和历史记录
- 支持按机型、状态筛选

纯前端实现，数据用 JSON 文件维护，不需要后端，部署在自己的服务器上，团队直接访问链接就能用。

## 用下来的感受

最大的收益不是「工具本身有多好用」，而是**把信息归拢到一个地方**这件事本身。

现在团队默认都来这里查，不再需要问「在哪个文档里」。Confluence 和石墨还在用，但版本状态这块有了一个明确的单一入口。

做这个工具本身花了不到两天，但它替代的是每次查信息要花的 5-10 分钟 × 每周好几次。
